## Overview

Pipeline v9.0 addresses quality issues discovered in test-gates (Logistics Network Simulator):

| Issue | Root Cause | v9.0 Solution |
|-------|-----------|---------------|
| Placeholder buttons (Edit, View, Help) | No story required them to work | UI element coverage + no-placeholder rule |
| Can't move nodes after placing | Missing completeness pair | Completeness pairs verification |
| No tutorial/onboarding | Not in scope | Onboarding question in Phase 1 |
| Missing edge cases | Insufficient test specs | Edge case matrix |
| General UX issues | No quality audit | 3-layer quality audit |

## Completeness Pairs

Actions that MUST be implemented together:

| Action | Required Pair |
|--------|---------------|
| Add / Create | Delete / Remove |
| Place | Move / Reposition |
| Connect | Disconnect |
| Open / Expand | Close / Collapse |
| Select | Deselect |
| Start | Stop / Pause |
| Show | Hide |
| Enable | Disable |
| Zoom in | Zoom out |
| Undo | Redo (or explicit "no undo") |
| Lock | Unlock |
| Pin | Unpin |
| Favorite | Unfavorite |
| Subscribe | Unsubscribe |

**Enforcement:**
- Phase 1: Verify stories exist for both halves
- Phase 2: Verify E2E tests cover both halves
- Phase 4: Verify implementation includes both halves
- Phase 5: Verify smoke test confirms both halves work

## No Placeholder Rule

STRICTLY FORBIDDEN:

```tsx
// Empty handlers
onClick={() => {}}
onDragStart={() => {}}
onChange={() => {}}

// Console.log placeholders
onClick={() => console.log('TODO')}

// Alert placeholders
onClick={() => alert('Not implemented')}

// Buttons without handlers
<button>Edit</button>

// Menu promises without delivery
aria-haspopup="menu" // with no actual menu
```

**Rules:**
- If it looks interactive, it MUST be interactive
- If it's in the UI, it MUST do something
- If you can't implement it now, DON'T add the element

## Onboarding Levels

Decided in Phase 1 via AskUserQuestion:

### Full Tutorial
Components:
- Tutorial.tsx - Main overlay controller
- TutorialStep.tsx - Individual step display
- TutorialHighlight.tsx - Spotlight on UI elements
- tutorialStore.ts - Step tracking, completion

Features:
- First-launch detection (localStorage)
- Step-by-step guide with highlights
- Skip button
- Demo scenario in Help menu
- Re-launch from Help > Show Tutorial

### Minimal (Default)
Components:
- EmptyState.tsx - Guidance when no data
- Tooltip.tsx - Hover hints
- HelpDialog.tsx - Keyboard shortcuts

Features:
- Empty states with call-to-action
- Tooltips on key UI elements
- Help menu with shortcuts list

### None (Expert Users)
Components:
- HelpDialog.tsx - Keyboard shortcuts only

Features:
- Help menu with shortcuts list only

## Phase 2 Changes

New requirements:

1. **Real Interaction E2E Tests** - No API calls, only UI actions
2. **Completeness Pairs in E2E** - Test both halves together
3. **Edge Case Matrix** - Min 2 edge cases per E2E test
4. **Onboarding Test Specs** - If epic exists

### Forbidden in E2E specs:
- "Call store.addNode() to add a node"
- "Invoke Tauri command directly"
- "Set state via browser.execute()"
- "Dispatch synthetic mouse/keyboard events"

### Required in E2E specs:
- "Drag element from palette to canvas"
- "Click button with selector [data-testid='...']"
- "Type text into input field"
- "Press Delete key while node selected"
- "Drag node from position A to position B"

### Edge Case Matrix:
| Category | Edge Cases |
|----------|-----------|
| Empty state | No items, first item, single item |
| Boundaries | Min value, max value, at limit |
| Invalid input | Empty string, special chars, too long |
| Rapid actions | Double-click, spam clicks, drag cancel |
| Interruption | Action during loading, mid-drag escape |
| State conflicts | Delete while editing, move while drag |

## Phase 4 Changes

New rules:

1. **No Empty Handlers** - Grep check before completing
2. **Implement ALL UI Elements** - Every menu item, button, form
3. **Completeness Pairs Implementation** - Both halves required
4. **Fix Previous Epic Bugs** - Don't ignore, fix them

Self-check commands:
```bash
# Check for empty handlers
grep -rn "onClick={() => {}}" src --include="*.tsx"
grep -rn "onClick={() => console" src --include="*.tsx"
grep -rn "onDragStart={() => {}}" src --include="*.tsx"
```

## Phase 5 Changes

3-Layer Quality Audit:

### Layer 1: Automated (Gate)
- Empty handler detection (grep)
- Test-only code paths detection
- Accessibility audit (axe-core)
- Visual regression tests
- Design token compliance (lint)

### Layer 2: Smoke Test (Manual)
- Click EVERY button
- Click EVERY menu item
- Try EVERY drag operation
- Submit EVERY form
- Test keyboard navigation
- Test edge cases (double-click, rapid clicks, escape mid-drag)

### Layer 3: Nielsen Heuristics (Evaluation)
1. Visibility of system status
2. Match between system and real world
3. User control and freedom
4. Consistency and standards
5. Error prevention
6. Recognition rather than recall
7. Flexibility and efficiency
8. Aesthetic and minimalist design
9. Help users recognize and recover from errors
10. Help and documentation

Rating: ✅ Pass | ⚠️ Minor | ❌ Major

Output: docs/quality-audit.md
