Create a pure function module for 2x2 (or NxM) grid layout positioning.

Export interfaces:
```typescript
interface MatrixCell {
  row: number;     // 0-indexed
  col: number;     // 0-indexed
  items: string[]; // Items to place in this cell
}

interface GridLabel {
  axis: 'top' | 'left';
  index: number;   // 0 = first column/row
  text: string;
}

interface PositionedMatrixElement {
  id: string;
  type: 'cell' | 'label' | 'item' | 'axis-line';
  x: number;
  y: number;
  width: number;
  height: number;
  text?: string;
  row?: number;
  col?: number;
}

interface GridLayoutResult {
  elements: PositionedMatrixElement[];
  bounds: { width: number; height: number };
}

interface GridLayoutOptions {
  startX?: number;       // Default 100
  startY?: number;       // Default 100
  cellWidth?: number;    // Default 250
  cellHeight?: number;   // Default 200
  labelHeight?: number;  // Default 40 (for top labels)
  labelWidth?: number;   // Default 80 (for left labels)
  itemHeight?: number;   // Default 30
  itemPadding?: number;  // Default 10
}
```

Export function:
```typescript
function layoutGrid(
  cells: MatrixCell[],
  labels: { top: string[]; left: string[] },
  options?: GridLayoutOptions
): GridLayoutResult
```

Algorithm:
1. Create axis lines (horizontal line between top labels and cells, vertical line between left labels and cells)
2. Position top labels centered above each column
3. Position left labels centered to the left of each row
4. For each cell: calculate its bounding box, then stack items vertically inside with padding
5. Return all positioned elements

For a 2x2 with labels ["High", "Low"] on both axes:
```
              High        Low
         ┌─────────┬─────────┐
   High  │ cell 0,0│ cell 0,1│
         ├─────────┼─────────┤
   Low   │ cell 1,0│ cell 1,1│
         └─────────┴─────────┘
```

<task type="auto">
  <name>Task 2: Create cluster layout algorithm for affinity diagrams</name>
  <files>live-canvas-mcp/src/layouts/cluster.ts</files>
  <action>
Create a pure function module for cluster-based layout positioning.

Export interfaces:
```typescript
interface Cluster {
  id: string;
  label: string;
  items: string[];
}

interface PositionedClusterElement {
  id: string;
  type: 'cluster-box' | 'cluster-label' | 'item';
  x: number;
  y: number;
  width: number;
  height: number;
  text?: string;
  clusterId?: string;
}

interface ClusterLayoutResult {
  elements: PositionedClusterElement[];
  bounds: { width: number; height: number };
}

interface ClusterLayoutOptions {
  startX?: number;        // Default 100
  startY?: number;        // Default 100
  clusterPadding?: number; // Internal padding, default 20
  clusterGap?: number;     // Gap between clusters, default 40
  itemHeight?: number;     // Default 35
  itemWidth?: number;      // Default 180
  itemGap?: number;        // Gap between items, default 8
  maxClustersPerRow?: number; // Default 3
}
```

Export function:
```typescript
function layoutClusters(
  clusters: Cluster[],
  options?: ClusterLayoutOptions
): ClusterLayoutResult
```

Algorithm:
1. For each cluster, calculate height = label + (items.length * itemHeight) + padding
2. Calculate cluster width = max(itemWidth, label width) + padding
3. Arrange clusters in rows (max N per row, then wrap)
4. Each cluster is a rounded rectangle containing:
   - Label at top (bold/larger text)
   - Items stacked vertically below
5. Return all positioned elements

Visual:
```
┌─────────────────┐  ┌─────────────────┐
│  Category A     │  │  Category B     │
│  ─────────────  │  │  ─────────────  │
│  • Item 1       │  │  • Item 4       │
│  • Item 2       │  │  • Item 5       │
│  • Item 3       │  │                 │
└─────────────────┘  └─────────────────┘
```
