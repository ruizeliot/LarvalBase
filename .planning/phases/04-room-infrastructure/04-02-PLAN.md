   ```bash
   cd live-canvas-mcp/viewer && npm install socket.io-client@^4.8.1
   ```

2. Create `src/hooks/useSocketIO.ts`:
   - Import `io, Socket` from "socket.io-client"
   - Import types from server (copy relevant interfaces or use shared types)
   - Create hook `useSocketIO()` returning:
     - `isConnected: boolean`
     - `roomCode: string | null`
     - `isHost: boolean`
     - `sessionUrl: string | null`
     - `createSession: () => void`
     - `joinSession: (code: string) => Promise<boolean>`
     - `leaveSession: () => void`
     - `send: (message: object) => void` (for backward compat with existing code)

3. Implementation details:
   - Connect on mount: `const socket = io({ autoConnect: true, reconnection: true })`
   - Handle `session_created` event -> set roomCode, sessionUrl, isHost=true
   - Handle `session_ended` event -> clear room state, alert user
   - Handle `canvas_state` event -> callback to update app state
   - Handle standard disconnect/reconnect
   - `createSession` emits "create_session" with callback
   - `joinSession` emits "join_session" with code, returns promise resolving to success
   - `leaveSession` emits "leave_session"

4. Accept `onMessage` callback in options (like useWebSocket) for backward compatibility with existing message handling.

Note: Keep the existing useWebSocket.ts for now - apps can choose which to use.
