Add to the connection handler in socketio.ts:

1. Add `disconnecting` event handler (NOT `disconnect` - see RESEARCH.md Pitfall 2):
   ```typescript
   socket.on("disconnecting", () => {
     // socket.rooms still contains room memberships here
     const roomCode = socket.data.roomCode;
     if (!roomCode) return;

     const room = roomManager.getRoom(roomCode);
     if (!room) return;

     if (room.hostSocketId === socket.id) {
       // Host is disconnecting - terminate session
       console.error(`[Room] Host disconnecting, terminating session: ${roomCode}`);

       // Notify all guests BEFORE rooms clear
       io.to(roomCode).emit("session_ended", {
         reason: "host_disconnected",
         message: "The session host has disconnected. Session ended."
       });

       // Force disconnect all sockets in room (after notification)
       io.in(roomCode).disconnectSockets(true);

       // Clean up room from manager
       roomManager.deleteRoom(roomCode);
     } else {
       // Guest is disconnecting - just notify others
       socket.to(roomCode).emit("user_left", { socketId: socket.id });
       // Room cleanup happens via leaveRoom
     }
   });
   ```

2. Add `deleteRoom(code: string)` method to RoomManager in manager.ts:
   - Simply removes room from the Map
   - This is for forced cleanup when host disconnects

   ```typescript
   socket.on("disconnect", (reason) => {
     console.error(`[Room] Socket disconnected: ${socket.id} reason: ${reason}`);
   });
   ```

CRITICAL: Use `disconnecting` event to access socket.rooms before Socket.IO clears them.
