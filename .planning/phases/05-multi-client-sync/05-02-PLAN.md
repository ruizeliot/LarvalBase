First, install lodash.throttle:

```bash
cd live-canvas-mcp/viewer && npm install lodash.throttle@^4.1.1 && npm install -D @types/lodash.throttle
```

Create new hook: live-canvas-mcp/viewer/src/hooks/useCanvasSync.ts

The hook should:
1. Accept options: socket (Socket | null), roomCode (string | null), onRemoteUpdate callback
2. Maintain refs for:
   - lastBroadcastRef: Map<id, { version: number }> - tracks what we last broadcast
   - localElementsRef: ExcalidrawElement[] - current local state for merging
3. Set up socket.on('canvas_update') listener that:
   - Receives { elements: SyncableElement[], fromSocketId: string }
   - Merges remote elements with localElementsRef.current using mergeElements logic
   - Updates localElementsRef.current with merged result
   - Calls onRemoteUpdate(merged) to update UI
4. Implement broadcastChanges function:
   - Takes elements: ExcalidrawElement[]
   - Updates localElementsRef.current = elements
   - Finds changed elements (where version differs from lastBroadcastRef)
   - If no changes, return early
   - Emits socket.emit('canvas_update', { roomCode, elements: changedElements })
   - Updates lastBroadcastRef for each broadcast element
5. Create throttled version using lodash.throttle(broadcastChanges, 16, { leading: true, trailing: true })
   - 16ms = ~60Hz, perceptually smooth
6. Cleanup effect: socket.off('canvas_update') on unmount

Export the hook returning { broadcastChanges: throttledBroadcast, setLocalElements }.

Include mergeElements function inline (or import from shared location) - same algorithm as server:
- Higher version wins
- Same version: lower versionNonce wins
- New elements from remote are added

TypeScript types: Use ExcalidrawElement from @excalidraw/excalidraw/types/element/types.

SyncableElement type: Define locally matching server types (id, version, versionNonce, isDeleted, type, x, y, width, height, etc.).
