<task type="auto">
  <name>Task 2: Create element merge utility with version resolution</name>
  <files>live-canvas-mcp/src/sync/merge.ts</files>
  <action>
Create new file live-canvas-mcp/src/sync/merge.ts implementing the version-based merge algorithm from Excalidraw's collaboration pattern.

Import SyncableElement from ../rooms/types.js.

Implement mergeElements function:
1. Takes local elements array and remote elements array
2. Creates a Map<id, element> starting with local elements
3. For each remote element:
   - If id not in map: add it (new element from remote)
   - If remote.version > local.version: replace with remote
   - If remote.version === local.version AND remote.versionNonce < local.versionNonce: replace with remote (deterministic tiebreaker - lower nonce wins)
   - Otherwise: keep local
4. Return Array.from(map.values())

This ensures deterministic convergence: all clients applying the same merge will end up with identical state.

Export: mergeElements function and re-export SyncableElement type.

WHY versionNonce tiebreaker: When two clients edit the same element simultaneously and produce the same version number, the random nonce provides a deterministic way to pick a winner. All clients will pick the same winner (lower nonce), so state converges.
  </action>
