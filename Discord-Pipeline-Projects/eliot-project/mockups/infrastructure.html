Implementation Details

💾 CSV Cache Layer

  • Load all CSVs on server startup
  • Parse with Papa Parse (streaming)
  • ~150 MB in memory
  • Indexed by VALID_NAME + APHIA_ID
  • Genus/Family aggregations pre-computed

⚡ ISR (Incremental Static Regeneration)

  • Pre-render species pages at build
  • revalidate: 3600 (1 hour cache)
  • On-demand revalidation via API
  • Stale-while-revalidate pattern

🔌 API Routes

  • /api/species/[name] — full species data
  • /api/search — fuzzy name matching
  • /api/export — CSV download endpoint
  • /api/filter — AND-logic trait filtering

🎨 D3.js Charts

  • Growth curves with SVG animation
  • Frequency barplots (qualitative traits)
  • Dot-strip charts with error bars
  • Depth charts with water gradients
  • Spectral temperature color scale

Cache Implementation

// lib/data/cache.ts — CSV cache singleton import Papa from 'papaparse'; import fs from 'fs'; class DataCache { private cache = new Map<string, any[]>(); async loadAll() { const files = [ 'egg_db', 'growth_db', 'settlement_age_db', 'settlement_size_db', 'vertical_position_db', 'pelagic_juvenile_db', 'rafting_db', 'swimming_ucrit_db', 'swimming_iss_db' ]; for (const f of files) { const csv = fs.readFileSync(`database/${f}.csv`, 'utf-8'); this.cache.set(f, Papa.parse(csv, { header: true }).data); } } } // pages/species/[name].tsx — ISR config export async function getStaticProps({ params }) { const data = await cache.getSpecies(params.name); return { props: { data }, revalidate: 3600 }; }
